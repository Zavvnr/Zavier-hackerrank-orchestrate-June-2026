name: clean-code
description: Cleans code with appropriate formatting, structure, and comments. Use appropriate naming conventions. explain how code works with each other and keep them concise. Avoid unnecessary complexity and redundancy. Focus on readability and maintainability.
---

When explaining code, always include:

1. **Start with an analogy**: Compare the code to something from everyday life
2. **Write some initial comments**: Write initial comments that describe the purpose of the code and its main components. This helps set the stage for the explanation and provides context for the reader.
3. **Walk through the code**: Explain step-by-step what happens
4. **Highlight and repeat**: Repeat 2 - 3 times to reinforce understanding. Highlight key points and concepts to ensure they are clear.

Keep explanations conversational. For complex concepts, use multiple analogies.
