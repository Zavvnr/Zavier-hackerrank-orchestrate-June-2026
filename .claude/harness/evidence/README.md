# Evidence corpus (experience observability)

Generated by `/evaluate`. Do not hand-edit case reports — re-run `/evaluate` to refresh them.

Layout:
- `overview.md` — the entry point: total accuracy + macro-F1, a ranked table of failure clusters
  (root-cause bucket -> count -> suspected component), and the top 1-2 fixes to try next.
- `case_<id>.md` — one root-cause report per mismatched sample case, written by the `eval-debugger`
  subagent: predicted vs expected (differing fields), the deciding image(s), the root-cause bucket,
  and the proposed component + smallest edit.

Read `overview.md` first, then drill into the cases the top cluster points to. This corpus is the sole
input to `/evolve` — decisions start from distilled root causes, never from raw model logs.
